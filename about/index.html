某集团交楼管理信息系统
开发工具:IDEA + Mysql + Springboot + Maven + Git
责任描述:参与交楼信息系统的迭代开发,项目核心技术人物,架构重构者。
项目描述:
1.ems审批管理功能的开发,完成交楼对ems对接,使得分户问题在整个流程有个闭环.
2.组织项目成员一起完成交楼代码重构,重新设计了交楼各个模块的关系,剥离以前多个模块合并在一起的聚合代码.使用微服务框架开发,删除了以前多数据源的方式,模块剥离使代码和数据库的设计更少的冗余,引入redis来共享session,灵活使用nignx解决前端session共享的问题.
3.日志模块的重构,之前的日志模块很不清晰,线上排查错误非常困难,重新设计日志模块,使用mq的方式来来解决日志量会过大的问题.

5.数据库的读写分离的使用,对于大数据的表进行拆分历史表,解决数据压力特别大的情况.

某地政府公文交换系统
开发工具:eclipse + Oracle + Maven + SVN
责任描述:参与整个项目需求调研以及项目概要设计详细设计讨论与分析。参与编写核心代码的模块有公文管理，审核流程，电子印章制作。以及现场实施部署。
项目描述:
某地政府内部有多个不同的局域网，通过使用网闸把不同局域网相连，通过网闸向不同的局域网进行发送数据。公文交换系统包括单点登录，PKI/PMI安全认证，用户管理和公文管理以及电子印章的制作的。通过PKI/PMI的安全认证之后，然后跳转到单点登录系统，登录进去公文交换系统里面，进行公文管理，发送公文的时候需要盖章的地方进行盖章申请，对应的部门通过PKI/PMI安全认证之后进行审核盖章。盖章完成之后进行公文的发送。
项目特点:
1、不同局域网的之间的数据交互
2、PKI/PMI的安全认证，其实用电子U盾的安全认证。
3、电子印章的制作和权限控制。
4、采用COS的方式单点登录
基于Netty+Spring的TCP长连接ARPG网页游戏
开发工具:eclipse + MySQL5.7 + Maven + Ant + SVN
责任描述:负责游戏后端的开发，开发了势力官职、生存试炼、VIP等系统的开发和维护一些比较核心的模块，战力和排行榜等。开服时处理相关Linux部署和数据导入导出。
项目描述:
项目是以角色为主的网页在线战斗类游戏，项目周期比较长，内容模块也比较多，主要说说我负责的模块:
1、势力官职：玩家达到某个等级之后可以进入一个势力，每个势力里面有对应的官职，官职可以通过挑战获得。每周四开发势力大战，让每个势力的在线报名的玩家进入战场进行定时的相互战斗，最后生存者可以获取对应的奖励。
2、生存试炼：玩家每天都可以进入这个模块进行挑战，设计了九层塔，每层塔里面都有对应的NPC，每挑战对应的NPC，血量不恢复，通过这样的方式来考验玩家能挑战多少层塔，挑战过每层塔都有对应的奖励。
3、VIP：VIP就是根据玩家充值和频繁程度获得的，有分免费的VIP（1-3），充钱即刻达到VIP4以上，VIP过期之后还会掉落到VIP1-3之间，VIP等级越高，战力和各个奖励都会有所相加。
项目特点:
1、项目采用Netty的作为数据连接桥梁。
2、项目中的数据持久层采用异步处理Insert，Update，Delete的方式。
3、基于项目是用jar方式加命令的方式启动的，项目采用Ant脚本连接到Linux进行项目拷贝到内网并执行对应重启服务器的命令用来达到更新程序。
4、项目采用Python做脚本处理数据库的开服操作。
